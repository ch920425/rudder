Rudder publishes stable releases for the user-facing install path:

```sh
npx @rudderhq/cli@latest start
```

This changelog tracks stable releases only. Canary builds are published more frequently for validation and Desktop asset testing; use [GitHub Releases](https://github.com/Undertone0809/rudder/releases) when you need prerelease history.

## v0.4.0

Released: 2026-06-24

[GitHub Release](https://github.com/Undertone0809/rudder/releases/tag/v0.4.0)

### New Features

- Added Feishu integration setup sessions, inbound callbacks, outbound responses, source badges, and read-only Feishu-bound chats with explicit fork handoff for operator replies.
- Added chat conversation forks, Messenger custom groups, pinned groups, generated group titles, group icons, and active chat actions aligned with the Messenger sidebar.
- Added richer Library and runtime surfaces, including organization skills in Library, workspace backup version downloads, syntax-highlighted editing, PDF/CSV previews, and runtime startup context bundles.

### Improvements

- Improved local runtime reliability for Codex, Claude, Cursor, OpenCode, Pi, and embedded Postgres by isolating runtime homes, preserving operator home access where needed, and tightening auth and platform diagnostics.
- Improved Messenger, chat, comment, and issue workflows with clearer mention rendering, grouped-row read handling, safer composer references, queue steering, and stronger sidebar loading caps.
- Improved maintainer workflows with product logic registry guardrails, release changelog taxonomy, lifecycle routing guidance, acceptance verifier skill coverage, and more explicit bundled-skill documentation.

### Bug Fixes

- Fixed Feishu and external chat edge cases around read-only composer display, setup routing, source badge contracts, and forked chat retitling.
- Fixed runtime and Desktop reliability issues around embedded Postgres platform repair, runtime skill materialization, heartbeat instruction scoping, update handoff, and public install behavior.
- Fixed UI and workflow stability across Library navigation, markdown issue comments, hover previews, website metadata icons, agent run scenes, cost summaries, dashboard caches, and CI/e2e release checks.
