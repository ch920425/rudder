# Issue Intake To Completion

This is a composed workflow view. It cites owning contracts and must not define
new product rules.

- Goal/project reason: `ORG.GOAL.001`, `ORG.PROJECT.001`.
- Create issue and hierarchy/comment context: `ISSUE.WORKFLOW.001`,
  `ISSUE.HIERARCHY.001`, `ISSUE.COMMENTS.001`.
- Assign, checkout, mention, or review route:
  `ROUTING.ASSIGNMENT.001`, `ROUTING.CHECKOUT.001`,
  `ROUTING.ATTENTION.001`, `ROUTING.COMMENT.WAKE.001`,
  `ROUTING.REVIEWER.001`.
- Wake and run agent: `RUN.WAKEUP.001`, `RUN.ADMISSION.001`,
  `RUN.AGENT.UNIFICATION.001`, `RUN.EXECUTION.001`.
- Load instructions and context: `AGENT.INSTRUCTIONS.001`,
  `CONTEXT.RESOURCES.001`, `WORKSPACE.PROJECT.001`.
- Persist run evidence: `RUN.RESULT.001`,
  `CONTROL.RUN.INTELLIGENCE.001`.
- Enter review when required: `ISSUE.STATE.001`,
  `ROUTING.REVIEWER.001`, `REVIEW.DECISION.001`.
- Release, follow-up, or learning: `RUN.ADMISSION.001`,
  `REVIEW.CLOSEOUT.001`, `LEARNING.PROMOTION.001`.
